# Question: 
描述当用户通过文本框输入一些内容并点击submit按钮后在https://studies.cs.helsinki.fi/exampleapp/notes页面上创建一个新笔记的情况。

# Answer:
> 当用户在 [https://studies.cs.helsinki.fi/exampleapp/notes](https://studies.cs.helsinki.fi/exampleapp/notes) 页面输入 “33443hello” 并点击 **Save** 时：
>
> 1. 浏览器根据表单定义 `<form action="/exampleapp/new_note" method="POST">` 创建 HTTP POST 请求。
> 2. 请求体中包含表单数据：`note=33443hello`。
> 3. 服务器接收请求后，返回状态码 302 并在响应头 `Location: /exampleapp/notes` 中要求浏览器重定向。
> 4. 浏览器自动执行重定向，向 `/exampleapp/notes` 发送新的 GET 请求。
> 5. 服务器返回新的 HTML 页面。浏览器随后请求 `main.css` 和 `main.js` 文件。
> 6. 浏览器执行 `main.js`，该脚本再向服务器请求 `data.json` 获取包含新笔记的最新笔记列表，并将其渲染到页面中。

## 时序图
```
title Creating a new note on the Notes page

browser->server: HTTP POST https://studies.cs.helsinki.fi/exampleapp/new_note\nForm data: note=33443hello
note right of server: Server receives the new note data

server-->browser: HTTP 302 Found\nLocation: /exampleapp/notes
note right of browser: Browser automatically follows redirect

browser->server: HTTP GET /exampleapp/notes
server-->browser: HTML document

browser->server: HTTP GET /exampleapp/main.css
server-->browser: main.css

browser->server: HTTP GET /exampleapp/main.js
server-->browser: main.js

note right of browser: Browser executes main.js

browser->server: HTTP GET /exampleapp/data.json
server-->browser: JSON data with all notes (including the new one)

note right of browser: Browser updates the DOM to show the new note

```
